# Status Definition

This document defines the repository status labels used to describe listed projects.

## Canonical Format

- Use lowercase values in Markdown entries and JSON data.
- Allowed values are: `active`, `experimental`, `stable`, `inactive`, `archived`, `unknown`.
- When status cannot be verified confidently, use `unknown`.

## Status Labels

### Active

Maintained and recently updated. Use this when there is visible ongoing maintenance or recent activity.

### Experimental

Prototype, proof of concept, or early-stage implementation. Use this for projects that are promising but not yet mature.

### Stable

Usable and reasonably mature for its intended purpose. Use this when the project appears dependable for normal use.

### Inactive

Not recently maintained, but still potentially useful. Use this when a project still has value even if it appears dormant.

### Archived

Read-only or explicitly no longer maintained. Use this when the repository or maintainers clearly indicate archival status.

### Unknown

The status cannot be confidently verified from the available public information.

## Notes

When in doubt, prefer `unknown` over making assumptions.
