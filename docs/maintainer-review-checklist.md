# Maintainer Review Checklist

Use this checklist when reviewing project submissions, pull requests, or direct catalogue updates.

For Codex-led pull request reviews, use this together with `docs/manual-pr-validation-workflow.md`.

## Repository Fit

- [ ] The project is open source.
- [ ] The repository is public and reachable.
- [ ] The project has credible relevance to Madagascar.
- [ ] The entry fits an existing category.

## Metadata Quality

- [ ] The entry follows `docs/repository-entry-format.md`.
- [ ] Required fields are present.
- [ ] Status uses an allowed lowercase value.
- [ ] License information is clear or explicitly marked `unknown`.
- [ ] Madagascar relevance is concrete and understandable.

## Catalogue Integrity

- [ ] The repository is not already listed elsewhere in the catalogue.
- [ ] The Markdown entry and `data/repositories.json` stay aligned.
- [ ] No obviously misleading or promotional wording is present.
- [ ] Notes explain uncertainty where needed.

## Trust and Safety

- [ ] The project does not appear malicious or deceptive.
- [ ] The submission does not expose sensitive personal data.
- [ ] The repository is not only a commercial landing page without reusable code or data.

## Review Outcome

- [ ] Accept
- [ ] Request changes
- [ ] Reject

If rejecting or requesting changes, explain the reason clearly and point contributors to the relevant standard or criterion document.
