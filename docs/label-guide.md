# Label Guide

This document defines a practical starter label set for triaging submissions and contributions in Open Source Madagascar.

## Contribution Flow Labels

| Label | Purpose |
|---|---|
| `type/submission` | New project proposal or repository entry request |
| `type/docs` | Documentation-only change |
| `type/data` | Change to `data/repositories.json` or schema |
| `type/templates` | Change to templates or contributor workflow files |
| `status/needs-review` | Ready for maintainer review |
| `status/needs-info` | Blocked until the contributor provides more details |
| `status/accepted` | Approved for inclusion |
| `status/rejected` | Not accepted |
| `status/duplicate` | Duplicate of an existing issue or project entry |

## Category Labels

| Label | Purpose |
|---|---|
| `category/payments` | Payment and mobile money related work |
| `category/ecommerce` | E-commerce and catalogue related work |
| `category/delivery-addressing` | Delivery, addressing, and location systems |
| `category/education` | Education and school-related tools |
| `category/health` | Health and clinic-related tools |
| `category/agriculture` | Agriculture and cooperative tools |
| `category/public-services` | Citizen and public-service tools |
| `category/language-ai` | Malagasy language and AI resources |
| `category/culture-history` | Culture, archives, and heritage resources |
| `category/developer-tools` | Developer tooling and reusable libraries |
| `category/datasets` | Datasets and open data resources |
| `category/offline-first` | Offline and low-connectivity tools |
| `category/security-identity` | Identity, consent, and security-related work |

## Suggested Usage

- Apply one `type/*` label to describe the contribution kind.
- Apply one `status/*` label to show review state.
- Apply one or more `category/*` labels when the work maps to a catalogue category.

This label set is a recommendation for maintainers. The labels still need to be created in the GitHub repository settings.

These labels can also be applied after a manual Codex review to reflect the review outcome.
