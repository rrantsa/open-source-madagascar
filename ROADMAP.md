# Roadmap

This roadmap translates the PRD into practical repository milestones. The project starts as a curated catalogue and grows over time into a broader digital commons for Madagascar.

## MVP

Focus: establish a structured and maintainable catalogue inside the GitHub repository.

- Finalize the main project documentation.
- Create category files for the first catalogue sections.
- Define contribution rules and project metadata requirements.
- Add a project submission template.
- Add a machine-readable `data/repositories.json` catalogue.
- Curate the first 5 to 10 example repositories.
- Expand toward at least 20 listed repositories.

## Short-Term

Focus: improve curation quality and make outside contributions easier.

- Refine category definitions and project selection criteria.
- Add status definitions and review guidance.
- Create GitHub issue and pull request templates.
- Add more repositories across key Madagascar-focused categories.
- Invite early contributors from local tech communities and schools.

## Medium-Term

Focus: add automation and improve discoverability.

- Validate repository entries automatically.
- Check JSON structure and broken links.
- Add GitHub Actions for routine verification.
- Improve the data model for future reuse.
- Explore GitHub Pages or another lightweight searchable interface.

## Long-Term

Focus: evolve the catalogue into a reusable public resource for a wider ecosystem.

- Launch a dedicated website with search and filters.
- Add project badges, maturity indicators, and richer metadata.
- Publish standards, templates, starter kits, and reference implementations.
- Support community governance and maintainer roles.
- Build partnerships with universities, communities, NGOs, and institutions.
- Encourage new open source projects based on clearly identified gaps.

## Guiding Direction

- Start as a catalogue.
- Evolve into a commons.
- Keep the barrier to contribution low.
- Favor transparency, readability, and reusability.
